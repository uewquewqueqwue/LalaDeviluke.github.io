    Character card system

    We have added a character card system, they are fully customizable. You have a number of characteristics that can be freely customized, such as age, hair color, etc.

    • hair
    • eyes
    • race
    • age
    • growth
    • accessory
    • ability
    • item
    • clothing
    • image

    We can give our meaning to all of this, it doesn't make sense to write it beautifully or just call it a word. For example, the hair can be just pink, black or just any color, or described like "she had beautiful black hair like darkness" well, you get it. You can also put a character and a photo on your card.

    Usage examples all command branches

    Creation

    • Let's start with creating a character. card_create is such a long command, horror, I made it easier to cc after we enter the name of your character, in principle it does not matter what it will be. But still choose wisely! As a result, at the end we get the command l)cc Mashiro ( in my case ). In principle, it does not matter, but you can open the menu of your character c and we will see his name and characteristics, but all the data in it will not be filled in, that is, none. So let 's fill them in ( at least a little )

    Customization

    • Now edit. We can start editing with a simple command card_edit oh, we forgot, everything is short, ce, let's start choosing one of the characteristics, we chose all the same hair. At the beginning, our hair is none, let's fix it using the example and the beginning of this blog. l)ce hair she had beautiful black hair like darkness. Success, you already have the completed hair data, then you can continue in the same spirit, the above lists what data you can change, or with the command l)ce after it the list will be visible ( Setting up a photo immediately if it is not clear l)ce image url ).

    Remove

    • Well, in principle, that's all, if you don't need a character anymore or you want a new one, just delete the old l)cr everything, and if you suddenly need it, the full command will be l)card_remove))
